# Tablet Holder Implementation Details

Этот проект реализует параметрический держатель для планшета с креплением VESA.

## Специфика моделирования

### 1. Позиционирование адаптера (Recess Logic)
Адаптер и основной корпус стыкуются по принципу "паз-шип":
- **Recess Floor (Дно паза)**: Находится на высоте `Z=3.0` в корпусе.
- **Joint "adapter_mount"**: Установлен в точке `(0, 0, 3.0)`.
- **Adapter Joint "mount"**: Установлен на ПЕРЕДНЕЙ грани адаптера (`Z=t`).

### 2. Интегрированный козырек (Hood "Bucket" Design)
Козырек выполнен в виде глубокого "ковша", который является единым целым с боковыми стенками корпуса. Профиль боковой стенки имеет трапециевидную форму (расширение в верхней трети), что обеспечивает максимальную защиту от бликов и прочность конструкции.

### 3. Пружинный механизм (Spring-Loaded Slider)
- **Slider Clip**: Подвижная деталь, обхватывающая корпус ("сендвич"-конструкция). Шейка слайдера ходит в открытом снизу U-образном пазу длиной 40 мм.
- **Anchoring System**: В качестве анкеров для пружин используются головки болтов M5. Болты проходят сквозь VESA-адаптер и основной корпус.
- **Face Surface**: На лицевой стороне корпуса предусмотрены шестигранные ниши под потайные гайки M5, чтобы крепеж не выступал над поверхностью и не мешал планшету.

### 4. Половины корпуса
- **Стык**: Обе половины строятся зеркально от плоскости `X=0`.
- **Сборка**: В `assembly.py` обе половины просто фиксируются в `Location((0,0,0))`. Это исключает любые ошибки округления при стыковке половин.

### 5. Джойнты
- `adapter_mount`: Основной интерфейс для крепления VESA.
- `slider_start`: Точка отсчета для позиционирования пружинного зажима.

## Параметры (params.json)
- `tablet_w/h/t`: Габариты устройства.
- `wall`: Базовая толщина стенки (влияет на жесткость).
- `adapter_w`: Размер VESA-площадки.
